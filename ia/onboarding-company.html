Company setup

The first step of the guided first run: name the company and pick its size, because the size pre-selects a fitting first program on the next step. The onboarding is a linear, one-time wizard under five minutes, sequential depth, not navigational depth. Serves J6 and the opinionated onboarding. Inside the operator app, so noindex. Source of truth: ia/docs/pages/onboarding-company.md.

Defined here, reused by 3.0 to 3.2

The wizard shell

The onboarding steps share a minimal chrome, defined in-node because it is thin and used only by cluster 3: a slim header with the Brio mark and a "Step N of 3" progress indicator, a single centered column, and one primary action per step. It deliberately omits the operator app's four job tabs, so the first run is a focused line, not a place to wander. This is "sequential depth, not navigational depth" made concrete.

Mobile-first priority

Content blocks

Block 1 is the first screen on mobile. Order reasoned from the 360px baseline up.

  1. Wizard header: the Brio mark and the "Step 1 of 3" progress indicator.
  2. Welcome: a short, warm line with the time budget ("Let's set up wellbeing for your team. It takes about five minutes.").
  3. Company name: a single text field.
  4. Company-size selector: the size buckets from 0.6. The pivotal input; it pre-selects the first program on 3.1.
  5. Why we ask: one micro-line, "Your size helps us pick a first program that fits."
  6. Primary action: Continue, to 3.1.

Components used: 0.6 Canonical lists (company-size buckets). The wizard shell, company-name field, and size selector are thin in-node elements. No new global invented.

Private app zone

SEO block (noindex)

Company setup is a transactional, authenticated wizard step. It is not indexed and carries no schema. The in-app findability rules still apply.

robotsnoindex, nofollow
canonicalnone for search; app URL is /app/onboarding/company
schema.orgnone (transactional wizard)
hreflang / OGnone

In-app findability (not SEO, still specified)

  • exactly one H1 (the welcome heading) for the accessible document outline
  • the size selector is a labelled radio group, not color-only
  • the progress indicator states "Step 1 of 3" in text
  • tap targets at least 44px
