First program confirm

Wizard step 2: show the operator one program we picked based on their size, explain plainly what it runs, and let them confirm it or pick another. This is where the interpretive layer (E1) and the canonical 0.4 card make the first choice feel easy and right. Serves Main, J6, and item 2. Inside the operator app, so noindex. Source of truth: ia/docs/pages/onboarding-program.md.

Mobile-first priority

Content blocks

Block 1 is the first screen on mobile. Order reasoned from the 360px baseline up.

  1. Wizard header: the Brio mark and "Step 2 of 3".
  2. Pre-select loading (F1 Load1): a brief loading while the size-based recommendation resolves; a 0.4 skeleton holds the space.
  3. Heading: a warm, competence-building line ("Here is a first program we picked for your team").
  4. Recommended program card (0.4, recommended state): the canonical card with the "Recommended for your size" tag, category, title, cadence and duration.
  5. What it runs: a short expansion, what it does and how the check-in works.
  6. Interpretive line (E1): "It is a light, weekly check-in. You can change it any time."
  7. Actions: Confirm and continue (to 3.2), primary; Pick another (to 5.0), secondary.

Components used: 0.4 Program card (recommended + skeleton), 0.6 Canonical lists (category), the wizard shell from 3.0. No new global; 3.1 is the onboarding consumer of 0.4.

SEO block (noindex)

  • exactly one H1 (the heading) for the accessible document outline

  • the loading state is announced, not silent
