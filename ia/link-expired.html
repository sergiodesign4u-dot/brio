Invalid or expired link

The error partner of 2.2. When an operator clicks a magic or reset link that has expired or was already used, this state explains plainly what happened, reassures that the account is fine, and offers a one-tap way to get a fresh link. A stale link is never a wall. It reuses the auth-page shell and the 2.2 confirmation layout with an attention mark. It closes cluster 2. Transient and transactional, so noindex. Source of truth: ia/docs/pages/link-expired.md.

Mobile-first priority

Content blocks

Block 1 is the first screen on mobile. Order reasoned from the 360px baseline up. The whole point of this state is that it is never a dead-end.

  1. Pre-auth header: the Brio mark and a "Back to log in" link (a reduced 0.1 variant, no job tabs).
  2. Attention mark and heading: a black-and-white attention badge (an exclamation, not a check, so success and error are told apart without color) and "This link has expired".
  3. Explanation: a short, non-blaming reason ("sign-in links expire after about 15 minutes and can be used only once") plus reassurance that the account is fine.
  4. Primary action: "Request a new link", to 2.1 Log in. One tap out of the error.
  5. Secondary help: a quiet "Still stuck? Contact support" line, so even the edge case has somewhere to go.

Components used: a reduced 0.1 (pre-auth header), the auth-page shell from 2.0, and the 2.2 confirmation layout with an attention mark. No new global. Adaptive copy: the same structure serves the reset path; only the noun changes ("sign-in link" versus "password-reset link").

Transient error state

SEO block (noindex, nofollow)

Invalid or expired link is a transient error state reached only by clicking a stale link. It is not indexed and has nothing to crawl, the same treatment as 2.2.

robotsnoindex, nofollow (transient error state; unlike the 2.0 / 2.1 forms, which keep follow)
reached byclicking an expired or already-used magic / reset link; not a browse target
schema.orgnone
hreflang / OGnone

In-app findability (still specified)

  • exactly one H1 (the heading) for the accessible document outline
  • the error is conveyed in text and an attention mark, not by color alone
  • Request a new link and the support line are labelled controls
  • tap targets at least 44px
