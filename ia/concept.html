Concept map

States (empty / error / loading) are not here, they live in the flows and the matrix; page content and cards are the detail layer. Navigation Model

Global navigation is the shortest path for the primary persona to her main job, not "what to put in a menu." Each global item is an entry into a job-cluster. The owner and employee surfaces have no multi-tab nav by design.

Global navigation (operator app)

Global itemJob-cluster behind itWhy global
Dashboard (home)E1 + J1 + J2: the pulse signal (score + interpretation + next action)The recurring aha and the reason she logs in. Post-login home
ProgramsMain + J6: run wellbeing (library, pick, manage)The core "what do I run" job, entered repeatedly
TeamMain + item 3: invite + participation (aggregate)Entered on every new hire, the invite / roster surface
OwnerJ1 + J5 + S2 + item 8: prove it works + upgradeTop-3 operator job by importance and the paid-conversion path
Settings utilitysupporting + item 4: Plan, Privacy center, Notifications, AccountUtility cluster, corner entry, not a job tab

Trade-off: "Owner" is only ~monthly-frequency and could be a contextual "Share this result" action instead. It is kept global because the owner dashboard is the paid feature and J1 / J5 / S2 score 3 for Priya. Reversible if usage shows it is rarely a fresh entry.

Depth: taps to the main job for Priya

Path to a job-clusterTapsWithin 3?
Returning operator, pulse signal + next action (recurring main-job value)1 (Dashboard is the post-login home)Yes
Run / adjust a program (Main + J6)Programs (1), Program detail (2), Check-in setup (3)Yes, 3 at deepest
Invite team / roster (item 3)Team (1), roster on the same screenYes
Prove it works / invite owner (J1 + J5 + S2)Owner (1), Invite owner + upgrade (2)Yes
One intentional exception, not a violation: the first-run guided onboarding is a linear wizard (company setup, first program confirm, invite team). This is sequential flow depth, not navigational depth. It is J6 itself ("tells me exactly what to do and in what order"), budgeted by time (under 5 minutes), runs once, and is not reachable from global nav afterward.

Global / contextual / deep

Global (always visible): the 4 job tabs (Dashboard, Programs, Team, Owner) plus Settings utility. Owner surface = single dashboard; employee surface = no nav.

Contextual (inside a flow): "Share this result with your owner" (on a positive milestone), "Set up check-in" (inside a program), "Resend invite" (inside roster), threshold-alert banner (on the Dashboard when it fires), pre-Q1 privacy disclosure (inside the check-in), upgrade prompt (on a free-tier limit).

Deep (rare): Plan / subscription change, Privacy center detail, Notification cadence, Account, and the one-time onboarding wizard.
