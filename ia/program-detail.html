Program detail / manage

The single program opened into a full page, where the operator reads what it runs and manages its lifecycle (finish setup, activate, end). The 0.4 card becomes the header, then expands into management. Serves Main and item 2. Inside the operator app, so noindex. Source of truth: ia/docs/pages/program-detail.md.

Mobile-first priority

Content blocks

Block 1 is the first screen on mobile. Order reasoned from the 360px baseline up.

  1. Nav + header card (0.1 operator, Programs active; 0.4 in header-summary form): category, title, cadence and duration, status badge. Back to 5.0.
  2. Status and primary action band: the one clear next step, which changes by status. Kept high so the non-expert always sees what to do.
  3. Overview / what it runs: one short paragraph, cadence, duration, category. Plain language.
  4. Check-in questions it carries: the 1 to 3 questions read-only, each with its type (emoji or slider); editing is 5.2, via "Edit check-in".
  5. Privacy reminder (0.3 operator variant, condensed): "You see aggregate only, never who answered." Reuse, not a new component.
  6. Manage actions: activate (draft to active), end (active to ended, opens the 0.5 confirm dialog), edit check-in (to 5.2).
  7. App shell footer (minimal): privacy link (1.1) and help.

Components used: 0.1 Navigation, 0.4 Program card (header-summary, status variants), 0.3 Privacy-disclosure (operator, condensed), 0.5 Dialog (End confirm), 0.6 Canonical lists. No new component; 5.1 becomes a new consumer of 0.5.

Private app zone

SEO block (noindex)

Program detail is inside the operator app, an authenticated per-account view. It is not indexed and carries no schema. The in-app findability rules still apply.

robotsnoindex, nofollow
canonicalnone for search; app URL is /app/programs/:slug
schema.orgnone (not a public landing page)
hreflang / OGnone

In-app findability (not SEO, still specified)

  • exactly one H1 (the program name) for the accessible document outline
  • status badge and questions are text, not color-only
  • manage actions are labeled controls; the End confirm follows the 0.5 dialog a11y contract (focus trap, Esc, inert background)
  • tap targets at least 44px
Reuse only. Program detail reuses 0.1, 0.4, 0.3, 0.5, and 0.6 and invents nothing; it adds 5.1 as a new consumer of the 0.5 dialog (End confirm). Next page: 5.2 Check-in setup (pick 1 to 3 questions, cadence, the minimum-N and no-live-count notes).
