Feedback and notifications

Three feedback surfaces, one for each job. Each is marked by shape and text, never by color alone.

Toast

Non-blocking, transient, auto-dismisses. For quick confirmations and light, recoverable failures (sent, saved, copied). Announced with aria-live.

✓

Invite sent

!

Could not send

Check the address and try again.

✓

Program ended

Inline message

Non-blocking but persistent, next to the thing that failed, keeping the form filled. For errors the user must act on in place, such as an invite-send or submit failure.

Some invites did not send. Your join link and code still work, so this never blocks you.

Dialog

Blocking and modal, until dismissed. For decisions and confirmations, such as ending a program, upgrading, or cookie consent.

End this program?

The team stops checking in for this program. You can start it again later.

When to use which

Toast Non-blocking, transient. Sent, saved, copied, and light failures.aria-live
Inline message Persistent, in the flow, next to the failure. Form stays filled.in place
Dialog Blocking, modal. Decisions and confirmations.role=dialog
